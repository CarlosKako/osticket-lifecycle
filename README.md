<h1>osticket-lifecycle Ticket Creation, Assignment, and Resolution</h1>
In this tutorial, we’ll practice creating, managing, and resolving tickets just like a real help desk. We’ll cover ticket properties, escalation, and best practices for handling requests from end users.<br />



<h2>Environments and Technologies Used</h2>

- Microsoft Azure (Virtual Machines/Compute)
- Remote Desktop
- Internet Information Services (IIS)

<h2>Operating Systems Used </h2>

- Windows 10</b> (21H2)

<h2>Ticket Lifecycle Objectives</h2>

- Create tickets as end users
- Observe ticket properties as help desk professionals.
- Modify ticket attributes such as Priority, Department, SLA, Assigned Agent.
- Work tickets to completion.
- Explore ticket visibility and access control.
- Practice real-world ticket intake and escalation scenarios.

<h1>🌍 Ticket Intake in Real Life (IRL)</h1> 
Tickets aren’t always created through the system directly. They may come via:

Phone call

Chat app (Slack, Teams, etc.)

Email

Web form

In-person encounters (“hallway requests”)

👉 Best Practice: Always create tickets for everything you do — even if you fix an issue on the spot.
Metrics (like response time and resolution rate) depend on it.
</p>
